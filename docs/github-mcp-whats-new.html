GitHub MCP Server
Latest Updates

The GitHub MCP Server has evolved fast since launch. Here's what changed — native HTTP transport, MCP Apps, tool filtering, expanded capabilities, and broader IDE support.

⚡
New Transport
Native Streamable HTTP
The server now includes native Streamable HTTP support via a new http command. The HTTP functionality previously hosted at api.githubcopilot.com/mcp now ships directly in the repo. Local stdio still works — this adds a remote option without a proxy.
// remote server — streamable HTTP "servers": { "github": { "type": "http", "url": "https://api.githubcopilot.com/mcp/" } }
🧪
Experimental
MCP Apps via Insiders Mode
A new opt-in mode for experimental tooling and features. MCP Apps was the first experiment — now migrated from insiders-only to a feature flag anyone can enable. Access the insiders channel for early features before they go stable.
// insiders channel — early access "url": "https://api.githubcopilot.com/mcp/insiders"
🔧
Configuration
Tool Filtering with --exclude-tools
New --exclude-tools flag and X-MCP-Exclude-Tools header let you shut off specific tools at the server level. Useful when you want Copilot to focus on issues and PRs but skip code search, or any combination that fits your workflow.
// disable specific tools $ github-mcp-server --exclude-tools search_code,list_commits // or via header (remote) X-MCP-Exclude-Tools: search_code,list_commits
📊
Expanded
CI/CD & Workflow Intelligence
The server now exposes tools for monitoring GitHub Actions workflow runs, analyzing build failures, managing releases, and getting pipeline insights. Previously you could manage code and issues — now you can manage your entire dev pipeline from Copilot Chat.
🔒
Security
Code Analysis & Security Findings
New tools for examining security findings, reviewing Dependabot alerts, understanding code patterns, and getting comprehensive codebase insights. The model can now surface vulnerabilities and suggest fixes without leaving the IDE.
💻
Client Support
Broader IDE & Client Compatibility
The remote server now works with VS Code 1.101+, Claude Desktop, Cursor, Windsurf, and other MCP-compatible hosts. Each handles config slightly differently but the same server URL works across all of them. PAT auth is universally supported; OAuth varies by host.
🐳
Infrastructure
Docker & Enterprise Improvements
Docker base images pinned to SHA256 digests for reproducible builds. Custom middleware support added. GitHub Enterprise Cloud with data residency supported via --gh-host flag. Stricter URL matching for github.com and ghe.com domains.
