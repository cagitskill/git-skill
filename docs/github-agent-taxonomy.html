GitHub Agent Terminology

// the naming map. use third-party agents as the umbrella, partner agents as the GitHub UI label, and keep extensions separate.

// The taxonomy

Three labels, one ecosystem

GitHub uses several names for things that look similar. Two are the same concept at different zoom levels. One is a different product entirely. Knowing which is which keeps customer conversations precise.

Umbrella category

Third-party agents

The canonical concept name in GitHub Docs for AI agents developed by external providers and supported alongside the Copilot cloud agent.

Examples
Anthropic Claude, OpenAI Codex.
Use when
Speaking generally, in customer decks, in reference docs.
Specific GitHub wording

Partner agents

GitHub uses "Partner Agent" in VS Code cloud session selection and "Partner agents" in org settings where admins enable supported third-party providers.

Where it appears
Cloud session dropdowns, policy pages, enablement settings.
Use as
A sub-label or parenthetical, not the broad category.
Separate concept

Extensions

GitHub Copilot Extensions expand Copilot with integrations and tools. Docs and community guidance distinguish them from agent terminology.

Use when
You mean plugins, chat integrations, or external tool connections.
Avoid when
You are describing autonomous coding agents that open PRs.
// Recommended phrasing

One sentence that lines up with the docs

Use this in decks and customer docs

"GitHub Copilot supports first-party Copilot agents, third-party agents from external providers, and custom agents defined in your own repositories. In some GitHub surfaces, supported third-party options are labeled as partner agents."

// Quick rule of thumb

When in doubt

Third-party → the category
Partner → the GitHub UI label
Extensions → a different product
Note on custom agents. Markdown-defined agents in your own .github/agents/ folder are a fourth category, covered separately in the Building Agents That Work deck. Custom agents are written by you; third-party and partner agents are authorized, not authored.
